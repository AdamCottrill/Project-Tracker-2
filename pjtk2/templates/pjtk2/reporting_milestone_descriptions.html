1.3 Project Proposal/Description Presentations

Project leads are required to make a brief presentation outlining the scope of each proposed project. A typical project description presentation does not need to be very long, 4-6 slides should be enough for a normal project. The presentations should include:

  • Project name and project number(s)
  • Study objectives - Why are we doing this? What question(s) are we hoping to answer?
  • Map(s) illustrating study area and access points
  • Methods used - especially if they are non-standard
  • Data
    • What are we going to collect? Why?
    • How will it be collected in the field?
    • How will it be stored? Where will it ultimately live?
  • Time frame - start and end dates, sampling frequency
  • If this is repeat project, what, if anything will be different this year.
  • Any other items that you think the rest of the staff should know.

The intent of the project proposals and associated presentations is to:

  1. Improve communication among staff
  2. Solicit input from other staff, and
  3. Provide an opportunity for pier-review of all our projects (at least to some degree).

1.6 Project Completion Presentations

Project leads are required to make a brief presentation outlining the basic results of each completed project. A typical project completion presentation does not need to be very long, 4-6 slides should be enough for a normal project, although the number of slides is project specific. Large projects with a lot of long-term data could have a dozen or more slides, while very simple projects may have only 1 or 2.

The project completion presentations should include:

  • the project name and project number(s)
  • Study objectives - Why did we run this project? What question(s) were we hoping to answer?
  • Map(s) illustrating study area and access points
  • Methods used - especially if they are non-standard
  • Time frame - start and end dates, sampling frequency
  • Data
    • What did we collect?
    • Summary of Catch Counts
  • How do these results compare with similar projects in the past or in different locations?
  • Did everything go as planned? Are things you would do differently?
  • Any other items that you think the rest of the staff should know.

Here are some example completion presentations that could be used as templates (simple to complex):

  • Benthic Sampling
  • Shawanagaarea_ESTN_2009
  • OCFA EGB Walleye Assessment
  • 2009 Commerical Catch Sampling

2 Milestone Naming Conventions

Before the reporting milestones can be uploaded to project tracker, they must be named so that they are unique, unambiguous, and will not conflict with other projects or reporting milestones. Names like "Project Description.doc", "Project Completion.pdf" or "Schedule.xls" are not unique and should be avoided.

Whenever possible, the Milestone Reports should be named using the following conventions:

"PRJ_CD(s)"-"Where"-"ProjectName"-"MileStone.ext"

The follow table provides some examples:

File Name
LHA_IA09_257-PointeauBaril-SMIN-proposal.ppt
LHA_IA07_002-CapeRich-Index-Completion.pdf
LHA_IA09_028-NCCoop-SamplingGuide-Captain.doc
LHA_SC11_122-SouthBay-RovingCreel-Description.doc
LHA_IA09_257_PointeauBaril-SMIN-Completion.ppt
LHA_IA10_027_Watchers_Index.zip
LHA_IA09_F13F14_IroquoisMacGregorBayFLIN_Completion.ppt

When the project reporting milestone is associated with more than one project is best to concatenate the project suffixes together (as in the Iroquois Bay/McGregor Bay FLIN completion presentation in the example table). If there are too many project to concatenate, they can be summarized by an appropriate description (e.g. - LHA_IA09_SUM-OffshoreIndex-Summary.pdf). Please contact Steve Currie and Lloyd Mohr if there are situations that have not been addressed by these examples.
